4. Test Methodology

PASS/FAIL definition: A test PASSES when the system behaves as expected -- even if it rejects the document. An expected rejection is a PASS. A test FAILS only when the system does something unexpected (accepts what it should reject, or rejects what it should accept).

Each test has a defined expected outcome for each system. For example: sending a document with @id to Python -- Python rejects it. That's a PASS (expected rejection). The old label "FAIL" was misleading -- it just meant "rejected" not "something went wrong."
